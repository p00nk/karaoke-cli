## [v1.4.0] 2026-06-26 — Фикс нестабильного CTC при длинном вступлении

### Sync
- **Обрезка вступления перед CTC-выравниванием**: при длинном инструментальном вступлении
  (≥7с по LRC) WhisperX forced alignment нестабильно размещал первые слова в инструментале
  (например "Stay," на 0:37 вместо 0:59 при 55-секундном вступлении). Исправлено: аудио
  перед CTC-выравниванием обрезается до `first_lrc_time - 5с`, WhisperX больше не "видит"
  инструментального вступления, временные метки результата сдвигаются назад на величину
  обрезки. Логика не задействуется для треков с коротким вступлением (<7с).

## [v1.3.0] 2026-06-26 — Фикс внутристрочного инструментала + кэп заливки

### Sync
- **Фикс пропущенного счётчика при инструментале внутри строки**: WhisperX forced
  alignment иногда оставляет большую паузу (>5с) между словами ОДНОЙ LRC-строки
  (например `"Ой да во сне"` + 17с тишина + `"привиделось"`). Механизм `gap_acoustic`
  проверял только межстрочные зазоры — внутристрочный инструментал не замечался и
  счётчик не появлялся. Добавлено: при последовательном назначении слов, если зазор
  между соседними словами ≥5с, вставляется `_line_end` перед словом после паузы.
  Строка разбивается, зазор становится межстрочным, счётчик/проигрыш срабатывает.
- **Кэп заливки слова `_MAX_FILL=4.0`**: CTC forced alignment растягивает `end`
  последнего слова до начала следующей строки (заполняет тишину). Это давало жёлтую
  заливку на 17–20с у одного слова — выглядело как рассинхрон. Кэп ограничивает
  длительность заливки 4с; остаток превращается в `\k`-паузу, суммарный таймлайн
  ASS остаётся точным.

## [v1.2.0] 2026-06-26 — Фикс анимации перехода + масштаб счётчика

## [v1.0.0] 2026-06-26 — Первый стабильный релиз: fade-анимация + финальные фиксы

### Visual
- **Замена blur/sharp → fade in/fade out**: анимация появления и исчезновения (`\fad(in,out)`)
  полностью заменяет устаревший `\blur8\t(...)` эффект для всех событий (karaoke, upcoming,
  pre-first upcoming). Обе длительности 500мс; для коротких событий автоматически делятся
  пополам, чтобы fade-in и fade-out не пересекались.
- **`--version` флаг**: `python karaoke.py --version` выводит `karaoke-cli 1.0.0`

### Sync
- **Авто-упрощение названия трека для lrclib**: если поиск по полному названию не даёт
  результата, автоматически удаляются теги в скобках (пр. `(БИТОДЕЛЬНЯ prod.)`) и год выпуска,
  повторный поиск по упрощённому названию (пр. `Спасибо (БИТОДЕЛЬНЯ prod.) 2019 → Спасибо`)
- **WhisperX: перенос строк по заглавным буквам**: WhisperX сигнализирует начало нового
  предложения заглавной буквой. Добавлен `_line_end` маркер перед словами с заглавной буквы →
  строки каракое совпадают с фразами, а не нарезаются по 7 слов
- **`--force-whisperx` флаг**: пропускает lrclib, принудительно использует WhisperX для
  транскрипции (полезно когда текст на lrclib отсутствует, неверен или предпочтительнее
  распознавание)

## [v0.9.3] 2026-06-26 — Фикс счётчика при длинных паузах + параметр --bg для видео

### Visual
- **`--bg` работает для видео-режима**: если параметр указан при подаче видео (URL), исходное
  видео заменяется статичной картинкой — удобно для создания «аудио-каракое» из видеоклипов
- **Фикс отсутствующего обратного счётчика после рефрена**: WhisperX forced alignment
  продлевает конец последнего слова до начала следующей строки (CTC-артефакт — заполнение тишины).
  Из-за этого `gap ≈ 0` и счётчик не срабатывал. Добавлен `gap_acoustic` с кэпом 5с на слово —
  используется только для определения интерлюдий/счётчиков, не влияет на отображение karaoke.
- `_COUNTDOWN_MIN` снижен 10.0 → 7.0 с (дополнительная защита для пограничных случаев)

## [v0.9.2] 2026-06-26 — Фикс прогрессивного рассинхрона (последовательное назначение слов)

### Sync
- **Корневая причина рассинхрона**: LRC-файлы lrclib.net могут иметь систематически ранние
  временные метки (в Алиса «Всё решено»: +0.4с в начале → +15с к концу), несмотря на
  совпадающую общую длительность. Per-segment alignment принудительно растягивает слова
  на эти неверные окна — отсюда рассинхрон.
- **Новый алгоритм — последовательное назначение**:
  - Глобальный CTC-forced-alignment (один сегмент = весь трек) даёт точные временные метки
  - Слова назначаются LRC-строкам **по порядку** (N слов строке 0, M слов строке 1 и т.д.)
  - LRC-временные метки полностью **не используются** для позиционирования слов внутри строки
  - Работает при точном совпадении числа слов (Whisperx forced-alignment сохраняет текст)
  - Допуск ±6.7% (`max(3, total//15)`) для небольших расхождений токенизации
- **Fallback — time-window matching**: если числа слов разошлись выше допуска → старый
  алгоритм по LRC-окнам с равномерным распределением для строк без совпадений

## [v0.9] 2026-06-25 — Статичная двухслотовая анимация + фикс переноса строк

### Visual
- **Анимация полностью переработана**: убран `\move` (слайд), введены два фиксированных слота
  - Слот A (Y=680): чётные строки → karaoke; нечётные → upcoming
  - Слот B (Y=575): нечётные строки → karaoke; чётные → upcoming
  - Позиция каждого события задаётся тегом `\pos(640,Y)` — MarginV переопределяется
  - Upcoming появляется В ТОМ ЖЕ слоте, что и следующее karaoke → при переходе только смена цвета (серый→белый+заливка), нулевое движение
  - `kend_by_slot[]` предотвращает наложение событий в одном слоте
- **Фикс переноса строк**: `_WRAP_CHARS` повышен 44→50
  - Строки ≤50 символов: нет `\N` → одна строка в обоих режимах (upcoming и karaoke)
  - Строки >50 символов: `\N` в mid-point одинаково для upcoming и karaoke → одинаковый перенос
  - Устраняет несоответствие: «одна строка в upcoming, две строки в karaoke» для одного текста
- **TODO (backlog)**: добавить полупрозрачный backdrop под текстовую зону перед презентацией

### Форматы входных файлов (документация)
- Уточнены поддерживаемые форматы в `--help` и docstring:
  mp3, flac, wav, ogg, m4a, aac, opus, mp4, mkv, avi, mov (любой формат, читаемый demucs/ffprobe)
- WMA: работа зависит от torchaudio-бэкенда, не гарантируется

## [v0.7] 2026-06-25 — Фикс YouTube субтитров + lrclib-приоритет + авто-язык

### Диагноз (YouTube URL)
YouTube auto-generated субтитры для русских треков приходят WITHOUT word-level timing:
- `dDurMs=0`, нет `tOffsetMs` в `segs`  
- весь блок текста — одна строка с `\n` внутри  
- `\n` в ASS-диалоге разрывал формат → всё после `\n` терялось → каждая вторая LRC-строка исчезала

### Sync
- **YouTube субтитры удалены из основного пайплайна**: новый флаг `--use-yt-subs` для явного включения
  - Без флага `download_video` не скачивает субтитры вообще (быстрее, нет риска 429)
  - Основной путь теперь: lrclib+WhisperX → WhisperX-транскрипция
- **Проверка word-level timing** (для `--use-yt-subs`): если нет `tOffsetMs` → вернуть `None`
- **\n-маркеры** (для `--use-yt-subs`): `\n` внутри события → `_line_end` на предыдущем слове
- **Авто-определение языка**: `_detect_lang_from_lrc()` — если >30% кириллицы → `ru`; не требует `--lang ru` для русских треков

### Visual
- **Сплит длинных строк**: в `_group_words_to_lines` после основного разбиения — жадный сплит групп >42 символов; предотвращает перекрытие Upcoming с двустрочным Karaoke

## [v0.6] 2026-06-25 — Редизайн анимации + счётчик кружков

### Visual
- **Анимация смены строк переработана**: теперь `Upcoming` владеет слайдом.
  - Upcoming появляется с blur→sharp, в последнюю секунду скользит вниз на позицию Karaoke
  - Karaoke стартует статично в финальной позиции — мгновенная смена цвета grey→white
  - Karaoke получает blur-out в конце, только если есть пауза после последнего слова
- **Счётчик кружков** (`●○○○` → `●●●●`) заменил числовой отсчёт (5,4,3,2,1)
  - Стиль `Countdown`: крупный шрифт, позиция над строками текста (Y≈520)
  - Исправлен off-by-one: последний кружок теперь `●●●●` (было `●●●○`)
  - Показывается перед первой строкой трека (если ≥2 сек до начала)
  - Показывается после пауз ≥10 сек (без ПРОИГРЫШ) и ≥20 сек (после ПРОИГРЫШ)
- **Два порога пауз**:
  - ≥10 сек: только кружковый отсчёт
  - ≥20 сек: `ПРОИГРЫШ XX сек` → blur-out → кружковый отсчёт

### Sync
- **Per-segment fallback в WhisperX alignment**: если сегмент не выровнялся — равномерное распределение для этого сегмента вместо потери строки (критично для быстрого рэпа)

## MVP-фокус (текущий спринт)

**Цель**: MP3 → видео-каракое, пригодное для презентации.

Критерии приёмки:
- Синхронизация текста с аудио: максимально точная
- Upcoming появляется заранее: 2–3 сек для медленных треков, 0.3–0.5 сек для быстрых
- Текст не прыгает, не моргает, не исчезает в середине строки
- Эстетичный внешний вид: шрифт, цвета, анимация
